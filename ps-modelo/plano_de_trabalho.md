
### **Plano de Trabalho – BikeToGo**

Este documento detalha as atividades necessárias para a conclusão do projeto. Note que são apenas conceitos primários que podem ser alterados até o momento final do projeto.

As atividades foram divididas em macro e microtarefas.

---

### **Atividade 1: Iniciação**
**Objetivo:** Definir e autorizar formalmente o projeto.

| Macrotarefa | Microtarefas |
| :--- | :--- |
| **1.1 Definição do Escopo** | 1.1.1 - Reunião de brainstorming para definir funcionalidades essenciais (MVP) e desejáveis. |
| | 1.1.2 - Redigir o Documento de Declaração de Escopo, listando o que está e o que não está incluído no projeto. |
| | 1.1.3 - Apresentar o escopo para a equipe e obter a aprovação de todos. |
| **1.2 Análise de Viabilidade** | 1.2.1 - Pesquisar tecnologias necessárias (linguagens, frameworks, banco de dados) e avaliar a capacidade técnica da equipe. |
| | 1.2.2 - Levantar recursos necessários (softwares, contas em serviços, etc.) e estimar o tempo de dedicação da equipe. |
| | 1.2.3 - Criar um breve relatório de viabilidade confirmando que o projeto é realizável com os recursos e conhecimentos disponíveis. |

---

### **Atividade 2: Planejamento**
**Objetivo:** Detalhar como o projeto será executado, monitorado e encerrado.

| Macrotarefa | Microtarefas |
| :--- | :--- |
| **2.1 Criação de Diagramas** | 2.1.1 - Criar o Diagrama de Classes para a arquitetura do sistema. |
| | 2.1.2 - Criar o Diagrama de Estados. |
| | 2.1.3 - Modelar o Diagrama Entidade-Relacionamento (DER) do banco de dados. |
| **2.2 Definição do Backlog** | 2.2.1 - Escrever as Histórias de Usuário (User Stories) para cada funcionalidade. |
| | 2.2.2 - Priorizar as histórias de usuário no Product Backlog. |
| **2.3 Cronograma** | 2.3.1 - Listar todas as atividades detalhadas em uma ferramenta (Ex: Trello, Planilha). |
| | 2.3.2 - Estimar a duração de cada atividade principal. |
| | 2.3.3 - Montar a linha do tempo do projeto (Ex: Gráfico de Gantt), definindo as datas de entrega de cada fase. |
| **2.4 Plano de Riscos** | 2.4.1 - Realizar uma reunião para identificar possíveis riscos (Ex: um membro ficar doente, uma tecnologia ser mais difícil que o esperado, etc.). |
| | 2.4.2 - Documentar os riscos e criar planos de mitigação simples para os mais prováveis/impactantes. |

---

### **Atividade 3: Execução**
**Objetivo:** Realizar o trabalho planejado para criar as entregas do projeto.

| Macrotarefa | Microtarefas |
| :--- | :--- |
| **3.1 Design (UI/UX)** | 3.1.1 - Desenhar os wireframes (rascunhos de baixa fidelidade) das principais telas do aplicativo. |
| | 3.1.2 - Criar os mockups (protótipos de alta fidelidade com cores e layout final). |
| | 3.1.3 - Apresentar o design para a equipe e fazer os ajustes necessários. |
| **3.2 Desenvolvimento** | 3.2.1 - Configurar o ambiente de desenvolvimento (repositório Git, IDEs, banco de dados). |
| | 3.2.2 - Desenvolvimento do Back-end: criar a base de dados, APIs e a lógica de negócio. |
| | 3.2.3 - Desenvolvimento do Front-end: implementar as telas e a interatividade com base no design aprovado. |
| | 3.2.4 - Integrar o Front-end com o Back-end. |
| **3.3 Testes** | 3.3.1 - Realizar testes unitários durante o desenvolvimento de cada funcionalidade. |
| | 3.3.2 - Executar testes de integração para garantir que o Front-end e o Back-end funcionam juntos. |
| | 3.3.3 - Realizar Testes de Aceitação do Usuário (UAT), onde membros da equipe (ou usuários externos) usam o app para encontrar bugs. |
| | 3.3.4 - Registrar e corrigir os bugs encontrados. |

---

### **Atividade 4: Monitoramento e Controle**
**Objetivo:** Acompanhar o progresso para garantir que o projeto permaneça no caminho certo. (Estas atividades ocorrem em paralelo com a Tarefa 3).

| Macrotarefa | Microtarefas |
| :--- | :--- |
| **4.1 Controle de Qualidade** | 4.1.1 - Realizar revisões de código (Code Review) entre os desenvolvedores. |
| | 4.1.2 - Validar se as entregas parciais estão de acordo com os requisitos definidos no escopo. |
| **4.2 Relatórios de Progresso**| 4.2.1 - Realizar reuniões rápidas de status (diárias ou semanais) para alinhar a equipe. |
| | 4.2.2 - Atualizar o cronograma e a planilha de status do projeto. |

---

### **Atividade  5: Encerramento**
**Objetivo:** Finalizar todas as atividades e formalizar a conclusão do projeto.

| Macrotarefas | Microtarefas |
| :--- | :--- |
| **5.1 Entrega do Projeto** | 5.1.1 - Preparar a documentação final do usuário e do sistema. |
| | 5.1.2 - Preparar a apresentação final do projeto. |
| | 5.1.3 - Realizar a entrega/apresentação final do Projeto. |
| **5.2 Lições Aprendidas** | 5.2.1 - Realizar uma reunião de retrospectiva para discutir o que deu certo, o que deu errado e o que pode ser melhorado em projetos futuros. |
| | 5.2.2 - Se possivel, criar algum documento listando as lições aprendidas e oque pode ser melhorado, e arquivar toda a documentação do projeto. |
